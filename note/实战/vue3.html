# 代码格式化 prettier (opens new window)

  1. 安装npm包
npm i prettier -D
1
  1. 在项目根目录新建配置文件，prettier.config.js，配置内容如下：
module.exports = {
  trailingComma: 'es5', //在逗号分隔的多行语法结构中，尽可能输出尾随的逗号
  tabWidth: 2, //指定每个缩进级别的空格数
  semi: true, //是否要分号
  singleQuote: true, //是否用单引号
  printWidth: 100, //单行字符长度大于多少时换行
};

# git commit 时检测代码

husky (opens new window)
你可以使用它来检测提交消息、运行测试、检测代码等等。Husky支持所有Git钩子。

git init
npx husky-init && npm install
1
2

执行完命令后，项目根目录下多了.husky文件夹，打开pre-commit，删除npm test,添加命令npx lint-staged。 下一步，在package.json里面添加配置如下：

"lint-staged":{
    "*.{js,ts,vue}":"eslint --fix"
}
1
2
3

lint-staged (opens new window) 一个对暂存的git文件运行lint的工具 是什么？该项目包含一个脚本，该脚本将运行任意shell任务，参数是由指定的glob模式筛选的暂存文件列表。
为什么使用？因为在提交代码之前运行代码检验更有意义。这样做，可以确保没有错误进入代码仓库，并且强制统一代码风格。但是在整个项目上运行各种代码检测是比较慢的。因此，我们只需要检查将要提交的文件。

最后，执行以下命令，测试配置是否生效。

git add .
git commit -m "feat: init project"
1
2

commitlint (opens new window) 检测git提交信息
安装.husky/commit-msg并在根目录添加配置文件commitlint.config.js

# Install commitlint cli and conventional config
npm install --save-dev @commitlint/{config-conventional,cli}
# Configure commitlint to use conventional config
echo "module.exports = {extends: ['@commitlint/config-conventional']}" > commitlint.config.js
1
2
3
4

在.husky/commit-msg文件中添加校验git提交信息的命令行

# Add hook
npx husky add .husky/commit-msg 'npx --no-install commitlint --edit "$1"'
1
2

然后，执行以下命令，测试配置是否生效。

git add .
git commit -m "add comment section"
1
2

结果显示不通过，改为如下符合规范的提交信息即可。

git commit -m "feat(blog): add comment section"
1

注意冒号后面必须有一个空格。
